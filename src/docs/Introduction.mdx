## What this is

This Storybook documents the **shadcn / Base UI** primitives in `src/components/ui/`. It mirrors the app’s styling (`globals.css`, design tokens) so you can preview components in isolation before using them in the Next.js dashboard.

| Section | Purpose |
| --- | --- |
| **[Design System](./?path=/docs/docs-design-system--docs)** | Full [`DESIGN.md`](https://github.com/RuiJieW/storybook-demo/blob/main/DESIGN.md) spec — colors, themes, typography, spacing, component rules |
| **UI** | One story file per component with variants and composed examples |

**Live deployments** (personal Vercel):

- App: [storybook-demo-pi.vercel.app](https://storybook-demo-pi.vercel.app)
- Storybook: [storybook-demo-ui.vercel.app](https://storybook-demo-ui.vercel.app)

## Run locally

```bash
npm install
npm run storybook
```

Open [http://localhost:6006](http://localhost:6006). The Next.js app runs separately:

```bash
npm run dev
# http://localhost:3000
```

Build a static Storybook (same output Vercel uses):

```bash
npm run build-storybook
# → storybook-static/
```

## Using the UI

### Sidebar

1. **Docs → Introduction** — this page  
2. **Docs → Design System** — token and layout reference from `DESIGN.md`  
3. **UI → …** — pick a component (e.g. Button, Card, Dialog)

### Theme toolbar

Use the **Theme** control in the top toolbar (paintbrush icon):

| Theme | When to use |
| --- | --- |
| **Light** | Default Clay cream canvas (`brand-light`) |
| **Dark** | Dark teal canvas (`brand-dark`) |
| **Wireframe** | Grayscale neutral — layout and structure without brand color |

Themes match `next-themes` in the app (`light`, `dark`, `wireframe`).

### Canvas vs Docs

- **Canvas** — interactive preview; resize the viewport with the layout controls.  
- **Docs** tab (per component) — autogenerated props table and story descriptions where configured.

### Controls & addons

- **Controls** — change props (variant, size, disabled, etc.) on supported stories.  
- **Accessibility** — a11y checks in the addon panel (configured as `todo` in CI).  
- **Interactions** — use stories with triggers (dialogs, menus, sheets) by clicking in the canvas.

## Adding or updating stories

Stories are colocated with components:

```
src/components/ui/button.tsx
src/components/ui/button.stories.tsx
```

After adding a component via shadcn CLI, generate a starter story:

```bash
node scripts/generate-ui-stories.mjs
```

Edit the generated file for realistic composition (e.g. Dialog + trigger + footer). Regenerate only for **new** files — it overwrites existing stories.

**Design doc page** loads `DESIGN.md` at build time via `src/docs/Design.mdx`. Edit `DESIGN.md` in the repo root; refresh Storybook to see changes.
